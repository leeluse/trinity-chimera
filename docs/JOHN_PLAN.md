## 기술적 상세 계획

### HMM Regime Classifier
- **알고리즘:** Hidden Markov Model 기반 시장 regime 분류
- **라이브러리:** hmmlearn, scikit-learn
- **입력:** OHLCV 데이터, 기술적 지표 (RSI, MACD, 볼린저밴드 등)
- **출력:** Bull/Sideways/Bear regime 분류 (3개 regime)
- **목표 정확도:** >85% backtest accuracy
- **모델 지속성:** pickle 또는 joblib을 통한 모델 저장/로드
- **실시간 예측:** 1시간 간격으로 regime 업데이트

### RL 트레이딩 환경
- **프레임워크:** Gymnasium (OpenAI Gym 호환)
- **관찰 공간:** 
  - 가격 데이터 (현재 가격, 이동평균, 변동성)
  - 포지션 상태 (롱/숏/홀드)
  - 잔고 및 자본 활용률
  - 기술적 지표 (10개 이상)
- **행동 공간:** 이산적 행동 {매수, 매도, 홀드}
- **보상 함수:** Sharpe Ratio + Drawdown 페널티
- **에피소드 길이:** 1000 time steps (약 42일 거래)

### 통합 인터페이스
- **데이터 포맷:** JSON 스키마 표준화
- **컴포넌트 통신:** ZeroMQ 또는 Redis Pub/Sub
- **모니터링:** Prometheus 메트릭 + Grafana 대시보드
- **에러 핸들링:** 재시도 로직 및 장애 복구 메커니즘
- **로깅:** 구조화된 JSON 로그 (로그 레벨별 분리)

### 테스트 인프라
- **단위 테스트:** pytest + coverage.py (목표 커버리지 >80%)
- **통합 테스트:** Docker 컨테이너 기반 end-to-end 테스트
- **성능 테스트:** locust를 이용한 부하 테스트
- **모의 거래:** paper trading 모드 지원
